# Weaviate 向量库 Web 客户端 — 产品与技术规格（Spec）

**版本**：0.1  
**技术栈**：Vue 3、Element Plus（Element UI 的 Vue 3 官方继任）、TypeScript（推荐）  
**目标**：在浏览器中连接 Weaviate，完成集群/元数据浏览、集合与对象管理、以及基于可配置 LLM 的向量检索。

---

## 1. 背景与目标

### 1.1 问题陈述

运维与研发需要在不依赖命令行的情况下，快速连接 Weaviate 实例、确认集群与集合健康状态、查看集合内对象与元数据，并对自然语言查询做嵌入后执行近邻检索。

### 1.2 产品目标

- 提供**单页应用（SPA）**，通过**连接地址（Host）、端口、鉴权**登录并持久化会话（安全策略见第 6 节）。
- 支持 **Weaviate 集群/节点层面的发现与只读查询**（健康、元信息、OpenAPI 能力等）。
- 支持 **向量检索**：用户输入自然语言或原始向量；自然语言路径需通过**可配置的嵌入模型（LLM/Embedding API）**生成向量再查询。
- 登录后主界面：**左侧为向量集合（Class/Collection）列表**；右侧为主工作区，支持查看集合**元数据与向量对象数量**、按**虚拟目录（文件夹）**浏览、以及**单条对象/文件级**查看。

### 1.3 非目标（首期不做）

- 不作为生产级多租户 SaaS（除非后续单独立项）。
- 不实现 Weaviate 集群的写入型运维（扩缩容、备份恢复 UI）——仅只读发现与查询为主。
- 不内置自托管模型推理服务；嵌入通过用户配置的 **HTTP API**（如 OpenAI 兼容接口）完成。

## 2. 术语

| 术语 | 说明 |
|------|------|
| 集合 | Weaviate 中的 Class / Collection（视客户端 API 版本而定）。 |
| 对象 | Weaviate 中存储的一条数据记录（含 `properties`、可选 `vector`）。 |
| 虚拟文件夹 | 由元数据字段（如 `path`、`directory`、`source`）推导出的层级浏览，非操作系统目录。 |
| 嵌入 | 将文本转为向量的过程；本工具通过外部 LLM Embedding API 完成。 |

## 3. 用户角色与典型流程

### 3.1 角色

- **访客**：仅可访问登录页。
- **已连接用户**：可浏览集群、集合、执行检索（受 Weaviate 与浏览器侧配置限制）。

### 3.2 主流程

1. 用户在登录页填写 **Host、端口、API Key/密码（若启用）**，可选 **HTTPS**。
2. 客户端调用 Weaviate 健康/就绪接口，成功则进入应用壳（Shell）。
3. 左侧加载**集合列表**；选中集合后加载 schema、对象计数（近似或精确，见 4.4）。
4. 用户可在「文件夹」视图中按路径字段聚合浏览；在「对象」视图中分页查看详情与向量摘要。
5. 在「向量检索」中配置嵌入提供方，输入查询文本，执行 `nearVector` / `nearText`（若启用模块）等查询并展示结果。

## 4. 功能需求

### 4.1 登录与连接

**输入字段（最小集）**

- **连接地址（Host）**：域名或 IP；**禁止**在规格中沿用错误用词，统一称「连接地址」。
- **端口**：数字，默认与协议匹配（HTTP 常用 8080，HTTPS 常用 443）。
- **密码 / API Key**：作为 Weaviate 的 `Authorization`（如 `Bearer <token>`）或等价头；若实例未启用鉴权，允许为空。

**行为**

- **连接测试**：提交前或提交后调用 `/.well-known/ready` 或 `v1/meta`（依版本）验证可达性与鉴权。
- **会话**：在内存或 `sessionStorage` 保存连接参数（**不建议**长期明文存 `localStorage`）；可提供「记住主机名、端口」开关，**密钥默认不持久化**。
- **错误提示**：网络错误、401/403、TLS 错误需可区分并给出可操作文案。

### 4.2 集群发现与查询（只读）

**能力**

- 展示 **节点/版本/meta**：从 `/v1/meta`（或 v4 客户端等价）读取版本、模块列表等。
- **健康**：`/v1/.well-known/ready`、`/v1/.well-known/live` 状态指示。
- **OpenAPI**：可选提供跳转或内嵌说明「当前实例 OpenAPI 文档 URL」，便于高级用户。

**约束**

- 所有请求经浏览器直连 Weaviate 时受 **CORS** 限制；规格要求文档中说明：**推荐**通过同域反向代理或开发环境代理解决跨域（见第 7 节）。

### 4.3 向量检索与嵌入配置

**嵌入配置（独立于 Weaviate 连接）**

- **Provider 类型**：至少支持 **OpenAI 兼容**（`baseURL` + `apiKey` + `model`）。
- **字段**：服务地址、模型名、API Key（敏感，不落盘或加密存储策略与登录一致）、可选 `dimensions`（若 API 返回固定维数可自动写入）。
- **操作**：「测试嵌入」按钮，用固定短句请求一次 embedding，校验维数与延迟。

**检索模式**

- **文本 → 嵌入 → nearVector**：主路径（不依赖 Weaviate 内置 `nearText` 模块时仍可用）。
- **可选**：若集群已配置 `text2vec-*` 等且可用，可提供 **nearText** 作为快捷方式（需检测模块能力）。

**结果展示**

- 返回对象的 **id、距离/得分、主要 properties**；可展开 **向量片段**（前 N 维）与完整 JSON。

### 4.4 登录后主界面：集合列表与内容浏览

**布局**

- **左侧边栏**：树或列表展示 **集合（集合列表）**，支持搜索过滤、刷新。
- **主区域** 选项卡或子路由：
  - **概览**：集合 **Schema**、**属性定义**、**对象总数**（`aggregate` 或 `/objects` 计数策略需与后端能力对齐）。
  - **文件夹视图**：用户可选 **用作路径的元数据字段**（如 `path`），按前缀/层级聚合为树；叶子可关联到对象列表。
  - **文件/对象视图**：表格分页列出对象；行点击打开详情抽屉：元数据、向量是否存在、原始 JSON。

**集合元数据与向量数**

- **元数据**：展示 class 的 property 定义、vectorizer、多租户配置等（只读）。
- **向量数/对象数**：优先 `Aggregate { meta { count } }`（GraphQL）或 REST aggregate；失败时降级提示并采用抽样说明。

## 6. 非功能需求

### 6.1 安全

- 密钥与 Token **默认仅内存**；任何持久化需用户明确同意并提示风险。
- 生产部署使用 **HTTPS**；避免在 URL 查询串传递密钥。
- XSS：对渲染的 JSON 做转义；避免 `v-html` 用户可控内容。

### 6.2 性能

- 集合列表与对象列表 **分页**；大对象 JSON **虚拟滚动**或折叠。
- 嵌入请求 **防抖**；检索并发限制。

### 6.3 可访问性与国际化

- Element Plus 组件保持键盘可操作；文案预留 i18n（首期可仅中文）。

## 7. 架构与技术要点

### 7.1 前端

- **Vue 3** + **Composition API** + **Pinia**（连接状态、当前集合、嵌入配置）。
- **Vue Router**。
- **Element Plus**（表单、表格、树、抽屉、消息提示）。
- **HTTP**：`axios` 或 `ofetch`；Weaviate 官方 **JavaScript/TypeScript 客户端**（推荐）封装 REST/GraphQL。

### 7.2 与 Weaviate 的交互

- 使用官方客户端版本与目标 Weaviate 版本矩阵一致（在 README 中锁定）。
- GraphQL 用于 aggregate、部分查询；REST 用于 schema、objects。

### 7.3 嵌入服务

- 浏览器直连第三方嵌入 API 可能遇到 CORS；**推荐**由同一后端代理转发（若首期纯前端，则必须在 Spec/README 中写明 CORS 限制与代理方案）。

### 7.4 跨域与代理

- 开发环境：`vite.config.ts` 中 `server.proxy` 将 `/weaviate` 代理到用户填写的 Host:Port。
- 生产：Nginx/Caddy 同域反代 Weaviate 与（可选）嵌入代理。

## 8. 验收标准（节选）

1. 填写正确 Host/端口/密钥后能进入主界面并显示版本信息。
2. 左侧能列出全部集合，与 Weaviate Console 大致一致。
3. 集合概览能看到属性与对象计数（在权限允许下）。
4. 配置嵌入后能完成一次文本检索并返回带分数的结果列表。
5. 对象列表能分页打开单条详情；文件夹视图在配置了路径字段时能展示层级结构。
